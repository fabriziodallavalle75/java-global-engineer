# Product Requirements Document (PRD)

Esta seção do Playbook descreve o processo de construção de um **Product Requirements Document (PRD)**.

O objetivo não é fornecer um template, mas apresentar um método para transformar a visão de um produto em um conjunto organizado de capacidades, funcionalidades e escopo de negócio.

O PRD descreve **o que o produto deve entregar**, mantendo a separação entre visão de produto, arquitetura e implementação.
